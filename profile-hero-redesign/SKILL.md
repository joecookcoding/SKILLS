# Profile Hero Redesign

A 5-step workflow that turns a clean-but-generic profile or personal-brand hero into a focused, conversion-driven layout. Each step produces inputs for the next; do not skip ahead.

## Core philosophy

Don't redesign for decoration. Redesign for **clarity, trust, direction, and action**. The target feel is: focused, modern, premium, credible, personal, conversion-driven, calm but confident.

## When to use

The user has a homepage / profile / newsletter / lead-magnet hero that:
- Looks clean but lacks energy or focal point
- Feels generic or template-like
- Has a weak/vague headline or invisible CTA
- Shows a portrait that feels placed rather than integrated
- Lacks trust signals near the CTA
- Doesn't communicate why the visitor should stay
- Reads fine on desktop but collapses on mobile

## Workflow

Run these in order. Stop after each step and confirm with the user before continuing — each step's output is an input the user should approve.

### 1. Hero Diagnosis → [HERO-DIAGNOSIS.md](HERO-DIAGNOSIS.md)
Find what is weak, what is working, what to preserve, what must change. Output: prioritized diagnosis that feeds step 2.

### 2. Message Hierarchy → [MESSAGE-HIERARCHY.md](MESSAGE-HIERARCHY.md)
Rewrite eyebrow, headline (one emphasized word), subheadline, value copy, CTA, and proof. Output: final hero copy block that feeds step 3.

### 3. Visual Composition → [VISUAL-COMPOSITION.md](VISUAL-COMPOSITION.md)
Layout, focal point, portrait integration, background depth, scan path, desktop + mobile composition. Output: design plan that feeds step 4.

### 4. CTA Conversion → [CTA-CONVERSION.md](CTA-CONVERSION.md)
Strengthen the CTA, button copy, microcopy, form fields, and proof placement. Output: conversion plan that feeds step 5.

### 5. Responsive Polish → [RESPONSIVE-POLISH.md](RESPONSIVE-POLISH.md)
Final desktop / tablet / mobile guidance, typography scaling, spacing, accessibility, and code-agent implementation notes. Output: handoff-ready plan.

## How to operate

When the user invokes this skill (or describes a matching hero problem), read the relevant step file in full before producing that step's output. Don't paraphrase from memory — the per-step files contain the exact prompts, output formats, decision rules, and quality checklists to follow.

If the user names a specific step ("just rewrite my headline", "I only need responsive polish"), jump directly to that file. Otherwise default to running step 1.

Pass the structured handoff fields between steps so each subsequent step has the inputs it expects.
