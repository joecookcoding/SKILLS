# Founder Validation Skill Pack

This pack contains five reusable `SKILL.md` files for moving from customer pain to a validated landing page and growth plan.

## Skills Included

1. **The Researcher**  
   Finds and validates painful customer problems before you build.

2. **The Strategist**  
   Turns validated pain into a high-value lead magnet.

3. **The Copywriter**  
   Writes hooks, benefit bullets, landing page copy, and CTAs.

4. **The Builder**  
   Turns the copy into a focused one-page lead magnet site.

5. **The Marketer**  
   Builds a 30/60/90-day funnel and growth plan.

## Recommended Workflow

```text
The Researcher
→ The Strategist
→ The Copywriter
→ The Builder
→ The Marketer
→ Back to The Researcher with real market feedback
```

## Operating Principle

Do not build until pain is validated.

Do not write copy until the lead magnet is clear.

Do not build the landing page until the copy is sharp.

Do not scale marketing until the page can capture leads.

Use each `SKILL.md` as a standalone agent instruction file, custom GPT instruction, Claude project instruction, Codex agent instruction, or internal workflow prompt.

Each skill includes:

- Purpose
- When to use it
- Required inputs
- Core prompt
- Output format
- Quality checklist
- Red flags
- Handoff instructions
