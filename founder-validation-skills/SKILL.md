# Founder Validation Skills

A 5-step workflow that takes a founder from raw customer pain to a validated landing page and growth plan. Each step earns the next.

## Operating principle

- Do not build until **pain is validated**.
- Do not write copy until the **lead magnet is clear**.
- Do not build the landing page until the **copy is sharp**.
- Do not scale marketing until the **page can capture leads**.

## When to use

The user is:
- Exploring a startup, SaaS, course, coaching, productized service, community, or newsletter
- Trying to figure out if real customers actually have the problem they imagine
- Choosing what free asset (checklist, calculator, template, guide, quiz) to ship
- Writing landing page copy for a lead magnet
- Building a one-page opt-in site
- Planning distribution, content cadence, or an email nurture sequence
- Looking for a 30/60/90-day plan tied to a real revenue goal

## Workflow

Run these in order. Stop after each step and confirm with the user — each step's output is the next step's input. The user can also enter at a specific step if earlier steps are already done.

### 1. The Researcher → [THE-RESEARCHER.md](THE-RESEARCHER.md)
Find and validate painful, urgent, repeated customer problems using Reddit, Amazon reviews, forums, app stores, review sites, and search signals. Output: 1–3 validated pains with exact customer language for step 2.

### 2. The Strategist → [THE-STRATEGIST.md](THE-STRATEGIST.md)
Turn validated pain into 5 ranked lead-magnet ideas (worth $50–$100, given away free) and select the winner. Output: lead magnet brief for step 3.

### 3. The Copywriter → [THE-COPYWRITER.md](THE-COPYWRITER.md)
Write hook → benefit bullets → landing page copy → CTA → optional follow-up email, in sequence, rooted in customer language. Output: full copy package for step 4.

### 4. The Builder → [THE-BUILDER.md](THE-BUILDER.md)
Build a focused, mobile-first, accessible one-page lead-magnet site (Next.js / Vite + Tailwind + shadcn defaults, email capture wired). Output: live landing page for step 5.

### 5. The Marketer → [THE-MARKETER.md](THE-MARKETER.md)
Funnel map + 30/60/90-day growth plan + content cadence + 5-email nurture + metrics + experiments. Output: measurable growth plan; results loop back to step 1.

## How to operate

When the user invokes this skill (or describes a matching founder problem), read the relevant step file in full before producing output — the per-step files contain the exact prompts, output formats, ranking rubrics, decision rules, and quality checklists to follow. Don't paraphrase from memory.

If the user names a specific step ("I just need lead magnet ideas", "write the email sequence"), jump directly to that file. Otherwise default to step 1 unless the user has clearly already validated pain.

Pass the structured handoff fields between steps so each subsequent step has the inputs it expects.
